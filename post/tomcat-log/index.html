Tomcat 日志处理

access log

通过 SpringBoot 配置，可以直接配置 tomcat access log，建议使用如下配置

 1

 3
 4

server:
  tomcat:
    accesslog:
      enabled: true # 打开 access 日志
      pattern: '%t %I %a %r %s, bytes sent: %b, take: %D us, %X'
      prefix: tomcat_access # 日志文件名
      suffix: .log # 日志文件后缀
      rotate: true # 按天写文件 tomcat_access.yyyy-MM-dd.log
      directory: /data/logs/${laix.config.name}/ # 文件存放路径，建议和项目log日志一致
      request-attributes-enabled: true #允许打印客户端IP, 协议,端口等
      buffered: false # 如果为 true 会立即刷盘，影响性能
      rename-on-rotate: true # 翌日写文件时才更新文件名为 yyyy-MM-dd

替代符含义如下（更全选项参见

https://tomcat.apache.org/tomcat-8.5-doc/config/valve.html#Access_Logging）

%t - Date and time, in Common Log Format
%I - Current request thread name (can compare later with stacktraces)
%a - Remote IP address
%r - First line of the request (method and request URI)
%s - HTTP status code of the response
%b - Bytes sent, excluding HTTP headers, or '-' if zero
%D - Time taken to process the request in millis. Note: In httpd %D is microseconds. Behaviour will be aligned to httpd in Tomcat 10 onwards.
%X - Connection status when response is completed:
X = Connection aborted before the response completed.
+ = Connection may be kept alive after the response is sent.
- = Connection will be closed after the response is sent.

快速配置

未接入新配置中心的项目直接在 bootstrap 配置文件中加入以上配置即可，同时保证有 laix.config.name 配置存在（对应项目简称）。

接入新配置中心的应用，直接引用共享配置 tomcat-v1 即可。

其他日志

container 上层日志

access log 的打印逻辑由 AccessLogValve 为 Tomcat Container 层 Valve 实现，只有才 web Service 层处理成功（逻辑成功）才会打印日志，如果请求没有进 Container 层，则根本不可能打印日志。

由于 springboot 已经统一了 tomcat 日志体系，可以通过设置包/类日志级别来打印更上层日志，如服务器开始发生 502 之类错误，通过配置 Http11Processor 类日志打印级别为 debug 即可。如下供参考

logging:
  level:
    #org.apache.coyote.http11.Http11Processor: debug
    org.apache.coyote: debug # 网络处理层
    org.apache.catalina.connector: debug # 连接层
    org.apache.tomcat.util.net: debug # socket 包装层

定制化的日志需求

springwebmvc 在 Servlet Filter 层提供了日志打印抽象 org.springframework.web.filter.AbstractRequestLoggingFilter，可以实现该抽象满足更具体的日志打印需要，如下实现可供参考

@Component
public class SimpleRequestLoggingFilter extends AbstractRequestLoggingFilter {
    private static final Logger log = LoggerFactory.getLogger(SimpeRequestLoggingFilter.class);

    @Override
    protected boolean isIncludeQueryString() {
        return true;
    }

    @Override
    protected boolean isIncludeHeaders() {
        return true;
    }

    @Override
    protected boolean isIncludePayload() {
        return true;
    }

    @Override
    protected boolean isIncludeClientInfo() {
        return true;
    }

    @Override
    protected void beforeRequest(HttpServletRequest request, String message) {
        getServletContext().log(message);
        log.info(message);
    }

    @Override
    protected void afterRequest(HttpServletRequest request, String message) {
        getServletContext().log(message);
        log.info(message);
    }
}
